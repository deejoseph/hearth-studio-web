# 工程级重构计划

## 1. 采用更清晰的分层架构
目标：`UI → Service → API Client → Backend`  
价值：减少页面直接调用 API 的耦合，统一错误处理。

## 2. 统一 API 标准与错误策略
目标：统一返回结构、错误码、异常处理。  
价值：前端可基于错误码做一致化处理，提升可维护性。

## 3. 抽象图片上传服务
目标：集中管理上传逻辑（大小限制、类型校验、进度回调）。  
价值：便于未来接入对象存储或多文件上传。

## 4. 引入可观测性
目标：接口日志、性能指标、错误上报。  
价值：提高问题定位效率，减少运维风险。

## 5. 模块化业务结构
目标：以 `features/order`、`features/auth` 组织组件与服务。  
价值：降低跨目录依赖，提高团队协作效率。

## 6. 数据层优化
目标：缓存高频接口、避免重复请求。  
价值：降低延迟，提高用户体验。
