# 性能分析

## 潜在瓶颈

### API 响应延迟
- 多数请求为链式读取（订单详情包含时间线、消息、图片）。
- 后端可能存在多表 Join 或 N+1 查询。

### 图片上传性能
- 上传使用 FormData，缺少分片/断点续传机制。
- 大图片会增加带宽与响应时间。

### 数据库查询
- 订单详情接口需要聚合多类数据。
- 订单看板接口在高订单量时可能压力较大。

### 前端渲染
- 时间线与订单卡片列表渲染可能造成大 DOM。
- 缺少虚拟列表与分页。

## 风险与优化方向
- 引入分页、过滤与按需加载。
- 后端为高频接口建立索引与缓存。
- 图片压缩与尺寸限制。
