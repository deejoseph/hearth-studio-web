# 未来改进建议

## API 标准化
- 统一返回结构（`success`, `data`, `message`, `error_code`）
- 统一错误码与错误处理策略
- 在 `api/client` 中实现统一拦截

## 错误处理
- 前端统一捕获并显示用户友好提示
- 增加全局错误边界

## 安全改进
- 后端接口增加权限校验
- 文件上传的安全策略与审计日志
- 管理端登录增加双因素或 IP 白名单

## 性能优化
- 页面级代码分割与懒加载
- 引入请求缓存策略（React Query/SWR）
- 图片懒加载与压缩

## 日志与可观测性
- 前端上报关键事件
- 后端记录关键 API 调用与错误

## 测试
- 核心流程端到端测试（下单、上传、进度）
- API 层单元测试

## 代码结构
- 按业务域聚合（features/order, features/auth）
- 把 API 调用从页面剥离到 service
