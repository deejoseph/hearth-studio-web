# 安全分析

## 认证
前端登录状态存储在 `localStorage` 中，安全性依赖后端接口的会话/权限校验。
建议：
- 后端使用 HttpOnly cookie 或 token 校验请求。
- 前端对敏感页面增加路由守卫。

## 授权
前端没有统一授权拦截层，依赖 API 响应判断。
建议：
- 在 API 层统一处理 401/403 并触发登出或跳转登录。
- 后端对订单详情、上传等接口做严格权限检查。

## 输入校验
前端有基本表单校验，但无法替代后端校验。
建议：
- 后端统一参数校验与错误返回结构。
- 前端在请求层对关键字段做简单检查。

## 文件上传安全
上传接口存在潜在风险：
- 文件类型与大小校验
- 恶意文件执行
建议：
- 后端限制 MIME 与扩展名
- 生成随机文件名
- 存储与访问隔离

## API 暴露风险
API 路径固定且可被直接调用。
建议：
- 增加频率限制与日志审计
- 对管理端接口增加额外校验
